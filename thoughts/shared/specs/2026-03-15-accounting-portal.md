# Бухгалтерский Портал (Accounting Portal) Specification

## Executive Summary
Полноценный full-stack веб-портал с лендингами для привлечения клиентов и закрытой частью (личными кабинетами) для действующего бизнеса (ИП и ООО). Цель портала — обеспечить быструю коммуникацию, 100% точность данных за счет интеграции с 1С и удобный обмен документами. Дизайн выполнен в строгом минималистичном стиле (черный, желтый, белый) с использованием 3D-анимаций.

## Problem Statement
Текущие процессы взаимодействия бухгалтерской компании с клиентами (ИП/ООО) требуют систематизации. Необходим единый цифровой центр (хаб), где клиенты могут быстро получать актуальные данные по налогам без ошибок, обмениваться первичной документацией, общаться со специалистами, получать новости и записываться на мероприятия, снижая нагрузку на менеджеров.

## Success Criteria
- 100% безошибочность отображения налоговых сумм и дат (за счет интеграции с 1С).
- Успешная и простая авторизация по номеру телефона без сбоев.
- Быстрая загрузка и скачивание клиентских документов любых размеров (интеграция с Yandex.Cloud S3).
- Бесперебойная работа портала при стартовой нагрузке (от 100 активных пользователей в день).

## User Personas
- **Руководители малого и среднего бизнеса (ООО) и Индивидуальные предприниматели (ИП)**. Ценят свое время, точность цифр и безопасность данных контрагентов.

## User Journey
1. Пользователь заходит на красивый лендинг портала (выполненный в черном, желтом и белом цветах, с 3D-анимациями).
2. Авторизуется в Личном кабинете по номеру телефона (получает SMS или код через звонок).
3. На главном экране (Дашборде) сразу видит:
   - Ближайшие налоги к оплате (суммы и даты автоматически подгруженные из 1С).
   - Кнопки для быстрой связи (чат с бухгалтером).
   - Последние загруженные/ожидающие документы.
4. Выполняет целевое действие: скачивает нужный счет, отправляет скан договора, читает статью, задает вопрос в чат или записывается на профильную конференцию.

## Functional Requirements
### Must Have (P0)
- Авторизация пользователей по номеру телефона.
- Личный кабинет клиента (ИП/ООО).
- Интеграция с базой данных 1С (выгрузка сумм налогов, актуального календаря взносов).
- Защищенный обмен документами (выгрузка файлов в Yandex.Cloud S3) с быстрой загрузкой/скачиванием.
- Встроенный чат со специалистом (консультации в реальном времени).
- Формирование простых счетов и актов прямо в интерфейсе кабинета.

### Should Have (P1)
- Система лендингов под каждую целевую аудиторию (ИП, ООО, специфичные ниши услуг).
- Раздел статей (блог, актуальные налоговые новости).
- Интерактивная система записи на конференции/вебинары.

### Nice to Have (P2)
- Push-уведомления или Telegram-уведомления о новых документах или сроках оплаты.
- Интерактивные 3D-анимации на лендингах для создания WOW-эффекта и повышения конверсии.

## Technical Architecture
### Data Model
- **User**: Клиенты компании (ИП/ООО) и Администраторы портала.
- **Document**: Файлы, загруженные клиентами или бухгалтерией (безопасные ссылки на Yandex.Cloud S3).
- **TaxRecord**: Записи о налогах и взносах (строго синхронизируются с 1С).
- **Invoice/Act**: Документы, сгенерированные пользователем внутри портала.
- **Message**: Сообщения во встроенном чате.
- **Article/Event**: Информационный контент для разделов новостей и конференций.

### System Components
- **Frontend**: React / Next.js (идеально для SEO лендингов и сложной логики личных кабинетов), библиотеки WebGL (напр. Three.js) для 3D-анимаций. Tailwind CSS / Vanilla CSS для кастомного черно-желто-белого минимализма.
- **Backend**: Node.js (Next.js API Routes или выделенный сервер Nest.js/Express) для обработки бизнес-логики и проксирования запросов в 1С.
- **Database**: PostgreSQL (надежное хранение структурированных транзакционных данных портала).

### Integrations
- **1C Enterprise**: Обмен данными (REST API 1С или OData) для достижения 100% точности.
- **Yandex.Cloud Object Storage (S3)**: Для безопасного и изолированного хранения всех файлов и документов.
- **SMS-шлюз**: Для обеспечения авторизации по номеру телефона.

### Security Model
- OTP-авторизация (One-Time Password) по SMS или звонку.
- Режим изолированного доступа: доступ к файлам в Yandex.Cloud S3 предоставляется только по временно подписанным (Signed) URL.
- Защита приложения (HTTPS, CORS, защита от CSRF/XSS).

## Non-Functional Requirements
- **Performance**: Мгновенный отклик интерфейса личного кабинета, плавная работа 3D-анимаций (60 FPS) на лендинге.
- **Scalability**: Базовая архитектура рассчитана на текущие 100 DAU, с легким масштабированием до тысяч пользователей при росте.
- **Reliability**: Автоматическое резервное копирование (бэкапы) базы данных. Автономия загрузки документов (S3).
- **Design**: Премиальный строгий минимализм (Цвета: Черный, Желтый, Белый). Интерфейс должен вызывать доверие к 100% точности.

## Out of Scope
- Полноценное ведение бухгалтерии пользователем внутри портала (портал служит хабом данных, сами расчеты происходят на стороне 1С).
- Разработка нативных мобильных приложений (портал будет адаптивным под мобильные браузеры (PWA)).

## Open Questions for Implementation
- Имеется ли у вашей 1С уже настроенный HTTP-сервис наружу, или настройку обмена с 1С нужно будет запланировать отдельным этапом с вашим 1С-программистом?
- Какой SMS-сервис предпочитаете использовать для отправки кодов (например, SMS.RU, СМС-Центр)?

## Appendix: Research Findings
- Подтверждена критическая необходимость использования Yandex.Cloud (S3) для избежания переполнения сервера сайта и сохранения наивысшей безопасности финансовых/личных документов клиентов.
- Выбран путь прямой интеграции с 1С как единственный инструмент, гарантирующий требование "100% безошибочности данных" (вместо ручного ввода через админку/Telegram).
